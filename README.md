# SQL-Data-Warehouse-Project

Hello and welcome to my **Data Warehouse and Analytics Project** repository.

This project is aimed as practice on how to build data warehouses from the ground up, and use analytics on the final product to generate insights from a business perspective. Its purpose is mostly to improve my understanding and skills, 
and also as a portfolio piece that should convey solid foundations and best practices from engineering to analytics.

For complete clarity, I'm using [Data with Baraa's guide](https://www.youtube.com/watch?v=SSKVgrwhzus) and the source of the data is also from him.

---

## Project Outline

### Building the Warehouse

#### Objective

Develop a warehouse using SQL Server that consolidates sales data, allows for efficient analytical reporting and insight generation.

#### Specifics

- **Data Source**: Data is imported from 2 source systems, ERP and CRM, stored as CSV files.
- **Data Quality**: Cleaning and resolving data quality issues before any analysis.
- **Integration**: Combine both data sources into a single model, convenient for querying and analytics.
- **Scope**: No historization - latest data only. 
- **Documentation**: Provide clear documentation on the steps taken to develop the warehouse and the data model.
